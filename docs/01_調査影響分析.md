## 1. 目的

ToDo管理画面の新規作成にあたり、対象リポジトリの現状、利用可能な技術スタック、既存資産への影響、およびリスクを整理する。

- リポジトリ: `uls-devin-sandbox/devin-test-kikuchi`
- 現在のブランチ: `main`
- 現状: `README.md` のみが存在するほぼ空の状態
- リポジトリの説明: `.NET Framework 4.8 の動作検証用の repository`
- 既存のプロジェクト: `origin/devin/1752072000-dotnet48-console-app` ブランチに `.NET Framework 4.8` コンソールアプリの試作例がある

| 項目 | 内容 |

| ターゲットフレームワーク | .NET Framework 4.8 |

## 4. 環境検証

- `.NET Framework 4.8 Runtime` はインストール済み
- `.NET Framework 4.8 Targeting Pack` は未インストールのため、ビルド時に `MSB3644` 警告が出る可能性がある
  - 実際の `HelloConsoleApp` ビルドでは警告が出たが、GAC から解決されて実行バイナリが生成された
  - 本 ToDo 管理画面でも同様の警告が発生する可能性があるが、動作には影響しない見込み
- 手動で `netfx-4.8-devpack` インストールすれば警告を解消できるが、本検証では不要と判断

## 5. 既存コードへの影響

- `main` ブランチには `README.md` 以外存在しないため、既存機能の変更・削除は発生しない
- `origin/devin/1752072000-dotnet48-console-app` は `main` 未マージの独立ブランチであるため、本 ToDo 管理画面の作成にあたり上書き・競合しない
- 新規プロジェクト追加のみで対応可能

## 6. 採用方針

- **UI 種別**: Windows Forms アプリケーション
  - 理由: `.NET Framework 4.8` でビルド・実行可能、追加ランタイム不要、サーバー環境でも GUI 表示が可能なため
- **アーキテクチャ**: 3 層構成
  - `TodoApp.Core`（クラスライブラリ）: ドメインエンティティ、永続化 I/F、ビジネスロジック
  - `TodoApp.WinForms`（WinForms EXE）: ユーザー画面、イベントハンドラ
  - `TodoApp.Core.Tests`（NUnit テストプロジェクト）: ビジネスロジックの単体テスト
- **データ永続化**: JSON ファイル (`tasks.json`)
  - 理由: DB セットアップが不要、単体テストでモックしやすい
  - 実装: `System.Runtime.Serialization.Json.DataContractJsonSerializer` を使用
- **単体テスト**: NUnit 3.14.0
- **外部ライブラリ**: NUnit のみ。業務アプリとして追加不要な NuGet パッケージは極力採用しない

## 7. 機能範囲（暫定）

- タスク一覧の表示（タイトル、期限、優先度、ステータス）

- タスクの削除
- タスクの完了/未完了切り替え
- ステータスフィルタ（全て/未完了/完了）
- JSON ファイルへの保存/読み込み

## 8. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| .NET 4.8 Targeting Pack 未インストール | ビルド警告（エラーではない） | 警告は無視し、実行確認を徹底。必要に応じて devpack インストール |
| NUnit テスト実行環境の未整備 | テスト実行ができない | `NuGet` 経由で `NUnit` および `NUnit.ConsoleRunner` を取得し、コマンドライン実行を確認 |
| Windows Forms サーバー環境での描画 | 画面確認が可能 | 本環境はデスクトップ GUI 有効。実装後に手動で起動確認 |
